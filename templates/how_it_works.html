System Story

Five stages from raw input
to performance signal.

Every prediction is the result of a disciplined pipeline — normalization, regression, classification, clustering, and optimization — all executing in parallel on three behavioral inputs.

Stage 1 — Data Ingestion

Behavior vectors enter the pipeline

The user provides three behavioral signals: weekly self-study hours (0–40), attendance percentage (0–100), and class participation (0–10). Each value is validated for range and completeness before proceeding.

study_hours Max: 40 h/wk
attendance Range: 0–100%
participation Scale: 0–10
Stage 2 — Feature Normalization

StandardScaler prepares the vector

A StandardScaler (fitted on the full training population) transforms raw inputs into zero-mean, unit-variance features. This is critical for both the distance-based clustering and the gradient-boosted models to operate on a level feature surface.

Scaler type StandardScaler
Fit source Training population
Output z-score vector
Stage 3 — Score Regression

Gradient boosting predicts the academic score

A gradient-boosted regressor takes the scaled feature vector and outputs a continuous score prediction in the 0–100 range. The model was trained to minimize RMSE and achieves an R² of {{ "%.4f"|format(meta.metrics.regression.r2) }} on held-out data, with a residual error of ±{{ "%.2f"|format(meta.metrics.regression.rmse) }} score points.

Algorithm GradientBoostingRegressor
R² {{ "%.4f"|format(meta.metrics.regression.r2) }}
RMSE {{ "%.2f"|format(meta.metrics.regression.rmse) }}
Stage 4 — Productivity Classification

A classifier maps behavior to a productivity tier

A gradient-boosted classifier assigns one of three productivity labels — High, Medium, or Low — and optionally returns per-class probability scores via predict_proba. The top probability feeds directly into the reliability band calculation. Overall accuracy: {{ "%.1f"|format(meta.metrics.classifier.accuracy * 100) }}%.

Algorithm GradientBoostingClassifier
Output classes High / Medium / Low
Confidence predict_proba
Stage 5 — Archetype Clustering

K-Means assigns a behavioral archetype

The scaled feature vector is compared against three cluster centroids using Euclidean distance. The nearest centroid determines the archetype — Consistent Learner, Night Owl, or Last-Minute Crammer. Each archetype carries a behavior profile, percentage distribution, and description.

Algorithm K-Means (k=3)
Assignment Nearest centroid
Archetypes 3 named clusters
Output Surface

Results are synthesized into an optimization surface

Score, productivity class, archetype, benchmark alignment, confidence reliability, behavior fingerprint, trend curve, and actionable suggestions are all computed from the above pipeline and returned in a single structured API response — ready for the mission-control UI to render.
